# MDS 改修工程表

MDS の拡張はフェイズ単位で進めます。
各フェイズでは、先に [mds-spec.md](./mds-spec.md) へタグ仕様を追加し、その後に実装と確認用 `.mds` の更新を行います。

| Phase | 対象 | 状態 |
| --- | --- | --- |
| Phase 1 | 表示系タグ、リンクタグ、画像表示、画像リンク | 実装済み |
| Include | MDS ファイルの相対パス展開、入れ子、エラー表示 | 実装済み |
| Phase 2 | フォーム系タグの表示仕様 | 未着手 |
| Phase 3 | フォーム入力値の保存、送信、イベント連携 | 未着手 |

## コンテンツ配置

- MDS の通常コンテンツは仮想 FS の `/var/www/mds/` 配下に置く。
- テスト用コンテンツは `/var/www/mds/test/` 配下に機能別ディレクトリを作って置く。
- テスト用画像などのアセットは `/var/www/mds/test/assets/` 配下に置く。
- `<img src>` などブラウザから取得するアセット URL は `/root/var/www/mds/...` を指定する。
- 自動確認では `/?mds=/var/www/mds/test/...&test=1` で `.eshrc` をスキップし、対象 MDS を直接起動する。
- `test=1` では CRT シェーダーを通さず素の terminal canvas を表示する。
- `test=1` では 1 フレーム 1 文字の出力制限、表示アニメーション、揺らぎ、ノイズ、scanline、curve、glass overlay をスキップする。

## Phase 1 詳細

状態: 実装済み

- 色指定は、以後指定のみを定義する。
- 下線、修正線、反転のテキスト装飾タグを定義する。
- `<a>` 形式のリンクタグを定義し、外部リンク、内部ページリンク、コマンドリンクを統一する。
- 画像表示タグを定義し、`left`、`center`、`right` の寄せを指定できるようにする。
- `<a>` の中に `<img>` を入れて、画像リンクとして扱えるようにする。
- 確認用画像は `/root/var/www/mds/test/assets/elegg-logo-163.png` を使用する。
- 機能別テスト MDS を `/var/www/mds/test/` 配下に追加する。
- `terminal.js` と確認用 `.mds` を更新済み。

## Include 詳細

状態: 実装済み

- `<include src="./parts/header.mds" />` 形式で別の MDS を記述位置へ展開する。
- パスは記述元 MDS からの相対パスとして解決する。
- `/var/www/mds/` 配下の `.mds` だけを対象にする。
- 入れ子を許可し、循環参照と読み込みエラーはページ内へ表示する。
- コードブロック内の `<include>` は展開しない。
- 確認用 MDS を `/var/www/mds/test/include/` 配下に置く。

## Phase 2 詳細

状態: 未着手

- テキストエリアを定義する。
- 1 行指定のテキストエリアはテキストフィールドとして表示する。
- チェックボックス、ラジオボタン、コンボボックスを定義する。
- このフェイズでは表示仕様を優先し、保存、送信、イベント処理は Phase 3 で扱う。

## Phase 3 詳細

状態: 未着手

- フォーム入力値の保持方法を定義する。
- 既存ボタンまたはコマンドリンクとの連携方法を定義する。
- ページ遷移後の状態維持が必要かを検討する。
