# AA Maker 仕様書

## 目的

AA Maker は、ブラウザ上でアスキーアートを編集する SPA として構築する。
80x25 の固定グリッド上に文字を配置し、半角文字と全角文字の表示幅を考慮しながら編集できることを目的とする。

## 対象範囲

この仕様書では、初期実装に向けた以下の範囲を定義する。

- アスキーアート編集用グリッド
- 左側ツールボックス
- 文字配置、範囲選択、削除、ハイライト移動、スポイト、ペン、テキスト入力
- キャラパレットセット、スタンプパレット、レイヤー
- 選択中文字・文字カラー・背景カラー表示
- 保存・読み込み・エクスポート機能の存在

保存・読み込み・エクスポートの詳細形式、スタンプの初期プリセット内容、Undo / Redo などは後続で定義する。

## グリッド仕様

### 基本

- 編集領域は 80 列 x 25 行のグリッドとする。
- 1 セルは半角 1 文字分の幅として扱う。
- 半角文字は 1 セルを消費する。
- 全角文字は横 2 セルを消費する。
- 全角文字のみで構成した場合、実質的な配置可能数は 40 列 x 25 行となる。
- 半角/全角の幅判定は、AA Maker が実際に使用するブラウザ上のフォント描画幅を DOM で測定した結果を基準にする。
- DOM 実測が使えない場合のみ Canvas 実測、さらに失敗した場合だけ `string-width` の結果を fallback とする。
- `Ambiguous` 幅の文字も Unicode 分類だけでは判断せず、実描画幅が全角寄りなら全角扱いにする。
- キャラパレット、Unicode 文字表、キーボード入力、グリッド描画は同じ幅判定を使う。

### 色の用語

背景色と文字色は以下の用語で扱う。

- `キャンバスカラー`: 未配置セルや `BGC:0` の背景表示に使う色。
- `FGDC` / `フォアグラウンドデフォルトカラー`: `キャンバスカラー` の反転色。
- `BGC`: 配置済みセルが保持する背景色。
- `FGC`: 配置済みセルが保持する文字色。

初期状態の `キャンバスカラー` は白 `ffffff` とする。
`キャンバスカラー` と `BGC` は別の値として扱う。
`キャンバスカラー` を変更しても、配置済みセルの `BGC` は変更しない。
`FGDC` は `キャンバスカラー` から常に算出し、`キャンバスカラー` が変わった場合は最新の反転色へ更新する。

### セル状態

グリッドセルは、キャラの有無と `FGC` / `BGC` の有無で状態を管理する。

- 未配置セルは、実データ上はキャラなしとして扱う。
- 未配置セルは、表示上は通常スペース `' '` として扱う。
- 特殊 SP を配置したセルは、`U+00A0 NBSP / no-break space` を持つ配置済みセルとして扱う。
- 特殊 SP は、見た目が空白でもキャラあり状態とする。
- キャラなし状態は `FGC:0, BGC:0` とする。
- キャラあり状態は `FGC:1, BGC:0` または `FGC:1, BGC:1` とする。
- `FGC:0, BGC:1` は存在しない不正状態とする。
- `BGC:0` の背景表示は `キャンバスカラー` を使う。
- `BGC:1` の背景表示はセルが保持する `BGC` を使う。

### セルの文字表示

セルの文字表示は以下のルールで常に表示する。

- 文字色はセルの `FGC` 指定に従う。
- 背景色は、セルに `BGC` 指定がある場合は設定された `BGC` に従う。
- 背景色は、セルに `BGC` 指定がない場合は `キャンバスカラー` に従う。

### 文字配置

- 各行は最大 80 セル分の幅を持つ。
- 文字を配置する際は、文字幅を判定してセル使用量を決定する。
- 全角文字を配置した場合、開始セルとその右隣のセルを占有する。
- 全角文字が 80 列目など残り 1 セルの位置に収まらない場合の扱いは、後続仕様で定義する。
- 既存文字と重なる配置の扱いは、ツールごとの操作仕様で定義する。

### 削除

- セル削除時は、そのセルに存在する文字を削除する。
- 全角文字の占有セルの片側を削除対象にした場合も、その全角文字全体を削除する。

## 画面構成

### 全体

画面は以下の領域で構成する。

- 上部メニューバー

- 中央編集領域

- 必要に応じて表示内容が変わる補助パネル

### 上部メニューバー

上部メニューバーには以下のメニューを配置する。

- `File` メニュー左側に、表示専用のアプリアイコンを配置する。
- `File`
- `Image`

ブラウザタブの favicon には、上部メニューバーのアプリアイコンと同じ画像を使う。

`File` メニューには以下を配置する。

- `Load`
- `Save`
- `Export`

`Load` / `Save` / `Export` の詳細仕様は、保存・読み込み・エクスポート仕様で後続定義する。

`Image` メニューには以下を配置する。

- `Invert BG`

`Invert BG` は `キャンバスカラー` を反転する。
この操作では `キャンバスカラー` に合わせて `FGDC` を更新する。
この操作では配置済みセルの `BGC` は変更しない。
編集エリアのセルの `FGC` が反転前の `FGDC` だった場合、最新の `FGDC` に変更する。
文字選択表示の `FGC` も、反転前の `FGDC` だった場合は最新の `FGDC` に変更する。

開発ビルドでは、上記に加えて `Dev` メニューを表示する。
`Dev` メニューは開発用のデータ生成や検証機能を配置する。
本番ビルドでは `Dev` メニューを表示しない。

### 左側ツールボックス

左側にツールアイコンを縦並びで表示する。
実装上のツールボックス幅は、アイコン列と選択中文字表示の実幅に合わせ、余分な横余白を持たせない。

ツールボックスには以下のツールを配置する。

- 範囲選択
- スポイト
- ペン
- 消しゴム
- テキスト
- スタンプ

左ツールボックスの最下部には、ツールアイコンとは別に選択中文字表示を配置する。

Penpot 参照では、上記に加えて反転系アイコンが 2 つ配置されている。
反転系アイコンの仕様は今回対象外とし、後続で定義する。

#### ツールアイコン

- 各ツールはアイコンで表現する。
- 実装時に、全ツール分の SVG アイコンを生成する。
- アイコン背景は白で統一する。
- アイコン本体は黒で統一する。
- 選択中のツールは、アイコン左側の縦線で選択状態を表現する。
- 選択状態は、背景色の変更ではなく左縦線で判別できるようにする。
- 移動ツールのアイコンは配置しない。
- アイコンサイズは Penpot 参照に合わせて実装時に調整する。
- ツールチップにはツール名とショートカットを表示する。
- ショートカットは、範囲選択 `V`、スポイト `I`、ペン `P`、消しゴム `E`、テキスト `T`、スタンプ `S` とする。
- `C` で選択中文字の文字色、`Shift + C` で選択中文字の背景色のカラーピッカーを開く。
- テキストフィールドやモーダル操作中は、ツールショートカットを無効にする。
- 選択中のツールアイコンを、マウスカーソルの右下 `+16,+16` に常時追従表示する。

### 選択中文字表示

左ツールボックスの最下部に、現在の選択中文字と色状態を表示する。

- 初期選択中文字は CP437 `DB` の `█` とし、コード表示は `CP437:DB` とする。
- 現在の選択中文字を表示する。
- 現在の文字カラーを表示する。
- 現在の背景カラーを表示する。
- 左クリックするとカラースキーマを開き、色を選択すると文字カラーに反映する。
- 右クリックするとカラースキーマを開き、色を選択すると背景カラーに反映する。
- 指定した文字カラーと背景カラーは、以後のペン・テキスト・スタンプ入力に適用する。
- 表/裏の切り替え仕様は持たない。

### 中央編集領域

中央編集領域に 80x25 の編集グリッドを表示する。

- 背景は白を基調とする。
- グリッド境界、ハイライト範囲、カーソル位置が識別できる表示にする。
- 左側ツールボックスと右側サイドバーに挟まれた主作業領域として扱う。

#### グリッドズーム

- `Ctrl + Wheel` で中央編集グリッドをなめらかにズームする。
- 通常のホイール操作はスクロールとして扱う。
- 論理グリッドは常に 80x25 のままとする。
- ズームで変更するのはセルの表示サイズのみとする。
- ズーム倍率は連続値で管理する。
- ズーム中心はマウスカーソル位置とする。
- 最小倍率と最大倍率を設ける。
- 実装時は `transform: scale()` だけに依存せず、倍率からセル幅・セル高を再計算する方式を優先する。
- macOS 向けに `Cmd + Wheel` 対応を検討する。

### 右側サイドバー

右側サイドバーは、選択中のツールに応じて内容を切り替える。

通常表示では以下を想定する。

- 情報表示
- キャラパレット
- キャラパレットセット選択用のコンボボックス
- レイヤーリスト

スタンプツール選択時は、キャラパレット領域をスタンプパレットに差し替える。
Penpot 参照では、最上部に情報表示、その下に `罫線1` のコンボボックス、中央にキャラパレット、下部に `名無し` 行のレイヤーリストが配置されている。

### 補助パネル

補助パネルは、色指定やスタンプなどの詳細操作で使用する。

- 選択中文字表示から色指定を行う場合は、カラースキーマを表示する。
- スタンプ選択時は、右サイドバー上部をスタンプパレットに切り替える。
- 必要に応じて下部または右下に表示する。

## 右側サイドバー仕様

### 情報表示

右サイドバーの最上部に、カーソル位置・ハイライト範囲・カーソル下セルの情報を固定表示する。

- カーソル下のグリッド座標を `X` / `Y` として表示する。
- `X` / `Y` は 0 始まりの座標とする。
- 範囲指定中またはハイライト表示中の場合、範囲の幅と高さを `W` / `H` として表示する。
- 未選択時の `W` / `H` は `--` と表示する。
- カーソル下セルのキャラをプレビュー表示する。
- キャラプレビューは、そのセルの文字カラーと背景カラーを反映して表示する。
- カーソル下セルの文字カラーを `FC:ffffff` の形式で表示する。
- カーソル下セルの背景カラーを `BC:000000` の形式で表示する。
- 色コードは 6 桁の小文字 16 進数で表示する。
- カーソル下セルが空白の場合、文字プレビューには灰色の矩形に斜線が入った SVG アイコンを表示する。
- カーソル下セルが空白で文字カラーが未設定の場合、`FC:------` と表示する。
- カーソル下セルが未配置セルの場合、背景カラー表示には `キャンバスカラー` を表示する。
- 例: `キャンバスカラー` が白の場合は `FC:------ BC:ffffff`、黒の場合は `FC:------ BC:000000` と表示する。

### キャラパレット

右サイドバー上部のキャラ選択領域をキャラパレットと呼ぶ。

- コンボボックスでキャラパレットセットを選択する。
- Penpot 参照の `罫線1` セレクトは、キャラパレットセット選択用のコンボボックスとして扱う。
- キャラパレットセットには、通常キャラパレット、ヒストリーパレット、キーボードインプット、Unicode 文字表を含める。

#### 通常キャラパレット

- パレットエリアは半角文字の場合 `16x16` セルで構成する。
- 全角文字のみのパレットセットの場合は `8x16` セルで構成する。
- キャラパレット自体は色を持たない。
- キャラパレットの文字色は `FGDC` で表示する。
- キャラパレット上の文字をクリックすると、その文字を選択文字としてセットする。
- セットされた文字は、左ツールボックス最下部の選択中文字表示に反映する。
- キャラパレットクリックで選択文字が変わった場合、ヒストリーパレットの履歴更新対象とする。

#### パレット表示ルール

パレットの文字表示は以下のルールで扱う。

- スタンプなど、セルとして表示する文字はセルの文字表示ルールに従う。
- セル以外の文字表示では、文字色は `FGDC` に従う。
- セル以外の文字表示では、背景色は `キャンバスカラー` に従う。

#### ヒストリーパレット

キャラパレットセットの中に、選択履歴を扱うヒストリーパレットを含める。

- ヒストリーパレット全体は `16x16` セルで構成する。
- 全体を選択履歴エリアとする。
- 選択履歴は、キャラパレットクリックまたはスポイトで選択文字が変わったときに更新する。
- 同じキャラが連続して履歴に追加されないようにする。
- ヒストリーパレット内の文字をクリックした場合も、その文字を選択文字としてセットし、左ツールボックス最下部の選択中文字表示に反映する。

#### キーボードインプット

キャラパレットセットの 1 つとして、直接入力用のキーボードインプットを用意する。

- テキストフィールドを 1 つ表示する。
- テキストフィールドに直接文字を入力できる。
- 複数文字が入力された場合は、先頭の 1 文字を選択中文字として扱う。

#### Unicode 文字表

キャラパレットセットの 1 つとして、Unicode 文字表を用意する。

- ASCII から Unicode までの文字をスクロールして選択できる。
- ASCII 文字は横 `8x16`、Unicode 文字は `16x16` ドットとして扱う。
- 全文字をスクロール表示する。
- 全文字を扱うため、表示は仮想スクロールで実装する。
- 一定間隔で文字コード行を挟んで表示する。
- コンボボックスの下に検索テキストフィールドを表示する。
- 検索テキストフィールドに文字コードを入力すると、該当コード位置までジャンプする。
- 将来的には、`ハート` などの名称検索で該当文字までジャンプできるようにする。

#### 似た文字検索

キャラパレットセットの 1 つとして、似た文字検索パレットを用意する。

似た文字検索パレットは以下の要素で構成する。

- 最上部の検索テキストフィールド
- 検索ボタン
- キャンセルボタン
- 検索パラメータ
- 検索結果の文字パレット

検索テキストフィールドでは、入力文字列の先頭 1 文字を検索対象とする。
検索ボタンを押すと、検索結果の文字パレットをクリアして再検索を開始する。
検索中は、しきい値に合致した候補文字を見つけ次第、検索結果へ順次追加する。
キャンセルボタンを押すと検索を停止し、その時点までに追加された検索結果は残す。

検索パラメータは以下を持つ。

- フォント
- 比較サイズ
- しきい値
- 幅一致優先
- 最大件数

検索対象は、開発用 Unicode ページ走査で生成された表示可能ページリストを使う。
ページリスト上で `present` のページだけを対象に、検索時にページ内の文字を Canvas へ描画して比較する。
ページリストは `apps/aa-maker/src/data/unicode-glyph-pages.json` に配備する。
候補文字は検索 Worker から返った順に検索結果へ追加し、スコア順の並び替えは行わない。

### 開発用 Unicode ページ走査

開発ビルドでは、似た文字検索用の表示可能 Unicode ページリストを生成できる。

第 1 フェイズでは、`Dev` メニューに `Scan Unicode Pages (All)` を配置する。
この処理は `0x20` から `0x10FFFF` までの全 Unicode 範囲をページ単位で走査する。

走査は以下のルールで行う。

- `1ページ = 256コードポイント = 16x16`
- ページ内で 1 文字でも Canvas に描画できる文字を見つけた場合、そのページは `present` とする
- `present` が確定したページは、残りのコードポイント走査をスキップする
- 1 文字も見つからないページは `absent` とする
- 連続する `present` / `absent` ページ数を RLE として JSON に保存する
- 完了後に JSON ファイルをダウンロードする
- 走査は Worker で分割して実行する

出力 JSON は以下の形式を基本とする。

```json
{
  "version": 1,
  "source": "aa-maker-canvas-glyph-page-scan",
  "pageSize": 256,
  "firstPage": 0,
  "pageCount": 4352,
  "range": {
    "start": 32,
    "end": 1114111
  },
  "scan": {
    "canvasSize": 16,
    "font": "16px \"MS Gothic\", monospace",
    "workerCount": 4,
    "notdefFilter": true,
    "startedAt": "2026-06-10T00:00:00.000Z",
    "finishedAt": "2026-06-10T00:00:01.000Z",
    "elapsedMs": 1000,
    "checkedCodePointCount": 1000,
    "presentPageCount": 10
  },
  "pageRuns": [
    { "startPage": 0, "present": true, "count": 1 },
    { "startPage": 1, "present": false, "count": 36 },
    { "startPage": 37, "present": true, "count": 14 }
  ]
}
```

### スタンプパレット

スタンプツールを選択した場合、右サイドバー上部のキャラパレット領域をスタンプパレットに差し替える。

- スタンプパレット上部に、スタンプセット選択用のコンボボックスを表示する。
- コンボボックスでスタンプセットを選択すると、その下のエリアにスタンプリストを表示する。
- スタンプリストは縦スクロール可能にする。
- スタンプリストの各項目は、スタンプ名と複数行 AA プレビューで構成する。
- スタンプデータには白黒スタンプとカラースタンプがある。
- スタンプツールへ切り替えただけでは、スタンプ選択やハイライト生成は行わない。
- スタンプリストでスタンプを選択すると、既存のハイライト種別に関係なく破棄し、選択したスタンプ contents のハイライトをグリッド中央に生成する。
- ハイライト生成時はスタンプの色ルールを反映する。
- スタンプがグリッド外にはみ出す場合も、ハイライト自体は生成できる。
- セルへの転写は、共通のハイライト確定操作で行う。

白黒スタンプの場合:

- `Invert BG` の影響を受ける。
- `キャンバスカラー` / `FGDC` の反転ルールに従って表示・配置する。
- `BGC` は保持する。

カラースタンプの場合:

- スタンプ自身の `FGC` / `BGC` を保持する。
- スタンプ内の透明または未配置部分だけ `キャンバスカラー` を反映して表示する。
- `Invert BG` によって変化するのは `キャンバスカラー` が見える部分のみとする。

### レイヤー

右サイドバー下部の `名無し` と表示されている各行を、1 つのレイヤーとして扱う。

- 各レイヤーは 80x25 のグリッドを持つ。
- レイヤー行をクリックすると、そのレイヤーが選択状態になる。
- 選択中のレイヤーのグリッドを編集対象とする。
- 非表示のレイヤーは選択できない。
- ロック中のレイヤーは選択できない。
- レイヤー行を上下にドラッグすると、レイヤー順を入れ替えられる。
- リストの下にあるレイヤーほど背面、上にあるレイヤーほど前面として扱う。
- 描画時は一番下のレイヤーから順に上のレイヤーを描画し、合成表示する。
- 上位レイヤーの空白セルは透過し、下位レイヤーの表示を残す。

#### レイヤー表示切り替え

- レイヤー行の左側に目玉アイコンを表示する。
- 目玉アイコンをクリックすると、開いた目と閉じた目をトグルする。
- 目が開いている場合、そのレイヤーを表示する。
- 目が閉じている場合、そのレイヤーを非表示にする。
- 非表示レイヤーは合成表示に含めない。
- 非表示レイヤーは選択できず、編集もできない。

#### レイヤーロック

- レイヤー行の右側に鍵アイコンを表示する。
- 鍵アイコンをクリックすると、開いた鍵と閉じた鍵をトグルする。
- 鍵が開いている場合、そのレイヤーを選択・編集できる。
- 鍵が閉じている場合、そのレイヤーを選択できず、編集もできない。

#### レイヤー作成・削除

- 右サイドバー最下部の左ボタンで新規レイヤーを作成する。
- 右サイドバー最下部の右ボタンで選択中のレイヤーを削除する。
- レイヤーは最低 1 つ必ず残す。
- 最後の 1 レイヤーは削除できない。
- 新規レイヤー名は `Layer 1` から始める。
- 以後、新規作成ごとに `Layer 2`, `Layer 3`, `Layer 4` のように番号を増やす。
- 削除済み番号は再利用しない。

## ハイライト仕様

ハイライトは、範囲選択、スタンプ配置、テキスト確定、貼り付けで生成される共通の編集対象とする。
従来の選択範囲、配置後選択、貼り付け後選択はすべてハイライトとして扱う。

### ハイライト生成

- 範囲選択ツールで範囲を指定した場合、範囲内セルの contents を持つハイライトを生成する。
- 範囲選択からハイライトを生成した時点で、選択元のセルはすべて一時的に空白にする。
- 範囲選択から生成したハイライトをキャンセルした場合、ハイライト表示をやめ、選択元のセルを生成前の状態へ復元する。
- 全角文字の一部が範囲に含まれる場合、既存仕様通り全角文字全体をハイライト contents に含める。
- スタンプを選択した場合、セルへ直接転写せず、グリッド中央にスタンプ contents のハイライトを生成する。
- `Ctrl+V` で貼り付けた場合、セルへ直接転写せず、マウスカーソル位置にクリップボード contents のハイライトを生成する。
- テキスト入力を確定した場合、セルへ直接転写せず、入力内容の範囲でハイライトを生成する。
- マウス位置が取得できない場合は左上 `(0,0)` にハイライトを生成する。

### ハイライト移動

- ハイライトの上にマウスカーソルを移動すると、マウスカーソルは移動カーソルに変わる。
- ハイライトは、マウスドラッグで移動できる。
- ハイライトは、矢印キーで 1 セル単位に移動できる。
- `Shift + 矢印` では 10 セル単位に移動できる。
- ハイライト移動中は、編集エリアのセル本体に影響を与えない。
- ハイライトはグリッド外へ自由に移動できる。
- 確定時にグリッド外へはみ出している部分は転写しない。

### ハイライト確定

- `Enter` キーを押すと、ハイライト contents を現在位置の下のセルへ転写して確定する。
- ハイライト以外のグリッドセルまたはグリッド内目盛り領域をクリックすると、ハイライト contents を現在位置の下のセルへ転写して確定する。
- 通常確定では、ハイライト contents の空白も転写対象とし、転写先セルを空白にする。
- `Ctrl + Enter` キーを押すと、ハイライト contents を透明転写して確定する。
- ハイライト以外のグリッドセルまたはグリッド内目盛り領域を `Ctrl + クリック` すると、ハイライト contents を透明転写して確定する。
- 透明転写では、`EmptyCell` は転写せず、下のセルを変更しない。
- 透明転写でも、`U+00A0 NBSP / no-break space` は文字として転写する。
- 確定後はハイライト表示を解除する。

### ハイライトキャンセル

- `Esc` キーを押すと、ハイライト表示を解除する。
- 範囲選択から生成したハイライトの場合、キャンセル時に選択元のセルを生成前の状態へ復元する。
- ハイライト中に行った fill、削除、色設定などの contents 変更は、キャンセル時に破棄する。

### ハイライト操作

- `Ctrl+C` キーは、表示中ハイライトの contents をクリップボードにコピーする。
- `[DEL]` キーは、表示中ハイライトの contents を削除する。
- ハイライト表示中にキャラパレットで文字を選択した場合、ハイライト contents をその文字で塗りつぶす。
- ハイライト表示中に `Shift` キーを押しながらキャラパレットで文字を選択した場合、ハイライト contents の未配置セルだけをその文字で塗りつぶす。
- ハイライト表示中の範囲カラー、サブメニュー、コピー、削除、fill、文字色設定、背景色設定は、すべてハイライト contents に対して作用する。
- ハイライト contents に対する操作は、確定するまで下のセルには反映しない。

## ツール仕様

### 範囲選択

グリッド上の矩形範囲を選択するツール。

- ドラッグで開始セルから終了セルまでの範囲を選択する。
- ドラッグ中は、右縦罫線の横に高さ、下横罫線に幅をリアルタイム表示する。
- ドラッグ完了時に、選択範囲からハイライトを生成する。
- ハイライト生成後の操作は、共通のハイライト仕様に従う。
- `Ctrl+A` キーでツールを範囲選択に切り替え、エディター全体を選択範囲にする。
- `Ctrl+A` キーで選択した場合も、エディター全体からハイライトを生成する。
- `Ctrl+V` キーでクリップボードの内容からハイライトを生成する。
- macOS では `Ctrl+Cmd+V`、macOS 以外では `Ctrl+Alt+V` キーで空白を透過扱いにしやすい貼り付けハイライトを生成する。
- `Ctrl+Z` キーで直前の編集操作を取り消す。
- `Ctrl+Shift+Z` または `Ctrl+Y` キーで取り消した編集操作をやり直す。
- 貼り付け位置は、マウス位置とする。
- マウス位置が取得できない場合は左上 `(0,0)` にハイライトを生成する。
- 全角文字の一部が選択範囲に含まれる場合、その全角文字全体をハイライト contents に含める。
- ツールを切り替えても、表示中のハイライトは解除しない。

貼り付けハイライトでは、クリップボード内の空白文字を `EmptyCell` として保持する。

- 通常確定では、空白文字の位置も転写し、転写先セルを空白にする。
- 透明確定では、空白文字の位置では既存セルを変更しない。
- 透明確定では、空白ではない文字だけを転写する。
- 空白文字は貼り付け位置の進行には含め、文字の相対位置は維持する。
- 確定後は、グリッド内に収まった転写範囲に合わせてハイライトを解除する。

#### 範囲選択サブメニュー

ハイライト表示中にグリッドを右クリックすると、範囲選択用のサブメニューを表示する。

サブメニューの表示文言は英語とする。

- `Clear Colors`
- `Text Color...`
- `Background Color...`

`Clear Colors` は、ハイライト contents 内のキャラありセルの色情報をクリアする。

- 未配置セルは対象外とする。
- 対象セルの `FGC` は `FGDC` に変更する。
- 対象セルの `BGC` は `BGC:0` に戻し、背景表示は `キャンバスカラー` を使う。
- `U+00A0 NBSP / no-break space` はキャラありセルとして対象に含める。

`Text Color...` は、既存のカラーピッカーを開き、ハイライト contents 内のキャラありセルの `FGC` を選択色に変更する。

- 未配置セルは対象外とする。
- `U+00A0 NBSP / no-break space` はキャラありセルとして対象に含める。

`Background Color...` は、既存のカラーピッカーを開き、ハイライト contents 内のキャラありセルの `BGC` を選択色に変更する。

- 未配置セルは対象外とする。
- `U+00A0 NBSP / no-break space` はキャラありセルとして対象に含める。

### 範囲カラー

ハイライト contents 内のキャラありセルに対して、文字色または背景色をまとめて指定するツール。

- ハイライト表示中のみ選択できる。
- ハイライト未表示時は選択できない状態にする。
- 左クリックするとカラーパレットを表示する。
- 左クリックで表示したカラーパレットから色を選ぶと、ハイライト contents 内のキャラありセルの `FGC` をその色に変更する。
- 右クリックするとカラーパレットを表示する。
- 右クリックで表示したカラーパレットから色を選ぶと、ハイライト contents 内のキャラありセルの `BGC` をその色に変更する。
- 右クリック側のカラーパレットでは「カラーなし」を選択できる。
- 「カラーなし」を選択した場合、ハイライト contents 内のキャラありセルを `BGC:0` に戻し、背景表示は `キャンバスカラー` を使う。
- 未配置セルは対象外とし、色状態を変更しない。
- `U+00A0 NBSP / no-break space` はキャラありセルとして扱い、色指定対象に含める。

### スポイト

マウスカーソル位置のキャラクターを取得するツール。

- クリックしたセルにある文字を取得する。
- 取得した文字を選択中文字に設定する。
- 全角文字の占有セルの右側をクリックした場合も、その全角文字を取得する。
- 空白セルをクリックした場合の扱いは後続で定義する。

### ペン

選択中の文字をグリッドに配置するツール。

- 左クリックで現在選択されている文字を配置する。
- 右クリックは一時的なスポイトとして動作し、カーソル位置の文字を取得する。
- 配置後のカーソル移動や連続描画の仕様は後続で定義する。
- 全角文字配置時は 2 セルを消費する。

### 消しゴム

クリックした位置の文字を削除するツール。

- 左クリックで対象セルの文字を削除する。
- 全角文字の占有セルの片側をクリックした場合も、その全角文字全体を削除する。
- ドラッグ中の連続削除を行うかは後続で定義する。

### テキスト

テキスト入力を行うツール。

- グリッド上の開始位置を指定して文字列を入力する。
- 日本語入力に対応する。
- 入力文字列は、半角 1 セル、全角 2 セルとして配置する。
- 入力を確定した時点ではセルへ直接転写せず、入力内容の範囲でハイライトを生成する。
- セルへの転写は、共通のハイライト確定操作で行う。
- 行末を超える入力の折り返し・切り捨て・拒否の扱いは後続で定義する。
- 改行を含む入力の扱いは後続で定義する。

### スタンプ

複数文字からなるキャラクターの塊を配置するツール。

- スタンプツール選択時、右サイドバー上部をスタンプパレットに切り替える。
- スタンプツールへ切り替えただけでは、スタンプ選択やハイライト生成は行わない。
- スタンプセットを選択し、スタンプリストから任意のスタンプを選択する。
- スタンプを選択すると、既存のハイライト種別に関係なく破棄し、選択したスタンプ contents のハイライトをグリッド中央に生成する。
- ハイライト生成時はスタンプの色ルールを反映する。
- スタンプがグリッド外にはみ出す場合も、ハイライト自体は生成できる。
- セルへの転写は、共通のハイライト確定操作で行う。
- スタンプの初期プリセット内容は後続で定義する。
- スタンプの保存・編集・追加方法は後続で定義する。

## 保存・読み込み・エクスポート

保存・読み込み・エクスポート機能は含める。

編集操作後にページをリロードまたはタブを閉じようとした場合は、ブラウザ標準の確認ダイアログで編集データの消失を警告する。
保存または読み込みが完了した場合は、この警告状態をリセットする。

### エクスポート

エクスポートは以下の形式に対応する。

- Plain Text
- ANSI
- MDS
- HTML

出力先は以下に対応する。

- Download
- Clipboard

各形式は、行末の通常スペースを削除して出力する。
全角文字の tail セルは出力しない。
未配置セルは通常スペースとして出力する。

ANSI / MDS は、連続する同一色範囲では追加の色タグや制御コードを出力しない。
色状態は行末でリセットせず、複数行をまたいで維持する。
出力末尾でだけ必要な色状態を閉じる。

HTML は単体で表示できる完全な HTML 文書として出力する。
AA 本体は 80x25 の固定セル CSS Grid として出力する。
フォントは `MS Gothic` / `ＭＳ ゴシック` を優先する固定幅フォントとする。
セルサイズはエクスポート時のズーム倍率に依存せず固定値とする。
全角文字や emoji は 2 セル分の幅で配置する。
右端から通常スペースを trim したうえで、同一行内の連続する同一色セルだけを 1 つの `<span>` にまとめる。

検討事項:

- 保存形式
- 読み込み形式
- 色情報を保存対象に含めるか
- スタンプ情報を保存対象に含めるか
